Pour accéder avec succès à n’importe quel point de terminaison de l’API développeur de Fapshi, vous devez inclure des **identifiants API valides** dans chaque requête. Ces identifiants vous authentifient et garantissent un accès sécurisé aux services Fapshi.

## Aperçu

Les identifiants API sont composés de :

- `apiuser` – votre identifiant utilisateur API unique  
- `apikey` – une clé secrète agissant comme un mot de passe

Ces deux paramètres doivent être inclus dans les **en-têtes** de vos requêtes HTTPS. En cas d’omission ou d’erreur, l’API renverra une erreur d’authentification.

## Comment utiliser les identifiants dans vos requêtes

Incluez le `apiuser` et le `apikey` dans l’en-tête de chaque appel API comme ceci :

**En-têtes :**

`apiuser: VOTRE_API_USER`

`apikey: VOTRE_API_KEY`

Cela s’applique aux environnements **sandbox** (test) comme **live** (production).

## Sécurité et Responsabilité

<Warning>
Votre combinaison `apiuser` et `apikey` équivaut à votre **nom d’utilisateur et mot de passe** pour l’API Fapshi. Ne les partagez jamais et ne les exposez pas dans du code exécuté côté client (par exemple, dans un script JavaScript sur le navigateur). Vous êtes **entièrement responsable** de toute utilisation abusive de vos identifiants.

Toute utilisation abusive des identifiants API entraînant des violations des [Conditions Générales d’Utilisation](https://fapshi.com/fr/termes-et-conditions/) ou de la [Politique de Confidentialité](https://fapshi.com/fr/politique-de-confidentialite) de Fapshi peut entraîner la suspension ou la désactivation définitive de votre compte.
</Warning>
